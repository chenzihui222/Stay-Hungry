# 研究工作流整合 - 使用说明

## 📋 功能概述

已将"研究报告"和"研究笔记"两个模块整合为完整的"研究工作流"：

```
阅读报告 → 一键创建笔记 → 自动关联报告
```

## ✨ 实现的功能

### 1. 研究报告页面 (report-viewer.html)

每个报告卡片增加了两个操作按钮：
- **📖 阅读** - 查看报告详情
- **✏️ 写笔记** - 跳转到研究笔记页面并自动创建笔记

### 2. 研究笔记页面 (research.html)

#### URL 参数解析
当从报告页面跳转过来时，URL 格式：
```
research.html?action=create-note&report-id=123&report-title=xxx&...
```

#### 自动功能
- ✅ **自动创建笔记** - 根据报告信息生成笔记
- ✅ **自动生成标题** - "研究笔记 - [报告名称]"
- ✅ **自动生成内容模板** - 包含报告信息和思考框架
- ✅ **显示关联报告** - 在编辑器上方显示关联的报告链接

## 🚀 使用流程

### 方式一：从报告创建笔记

1. 进入"研究报告"页面
2. 浏览报告列表
3. 点击报告卡片的"✏️ 写笔记"按钮
4. 系统自动跳转到"研究笔记"页面
5. 自动生成笔记模板
6. 补充您的研究思考
7. 点击"保存笔记"

### 方式二：查看已关联的笔记

1. 在笔记列表中，带有"🔗 关联报告"标识的笔记表示关联了报告
2. 点击笔记加载
3. 在编辑器上方可以看到关联的报告链接
4. 点击链接可以返回报告页面

### 方式三：取消关联

1. 打开已关联报告的笔记
2. 点击关联报告区域右侧的"×"按钮
3. 关联被取消（笔记内容保留）

## 📝 技术实现

### 关键函数

#### report-viewer.html
- `renderReports()` - 渲染报告卡片，添加"写笔记"按钮
- URL 参数构建 - 传递报告信息

#### research.html
- `checkUrlParams()` - 解析 URL 参数
- `createNoteFromReport()` - 从报告创建笔记
- `generateNoteTemplate()` - 生成笔记模板
- `displayLinkedReport()` - 显示关联报告信息
- `removeLinkedReport()` - 取消关联
- `showNotification()` - 显示操作提示

### 样式文件

新增样式类：
- `.report-actions-group` - 报告卡片按钮组
- `.create-note-btn` - 写笔记按钮
- `.view-report-btn` - 阅读按钮
- `.linked-report-info` - 关联报告信息区域
- `.linked-report-badge` - 关联报告标识
- `.note-linked-indicator` - 笔记列表中的关联标识

## 🐛 故障排查

### 问题：点击"写笔记"后没有自动创建

**检查：**
1. 是否已登录？
2. 查看浏览器控制台是否有错误
3. 检查 URL 参数是否正确传递

### 问题：关联报告不显示

**检查：**
1. 笔记是否正确保存？
2. 查看 localStorage 中笔记数据是否包含 linkedReport 字段
3. 检查 CSS 样式是否正确加载

### 问题：模板生成失败

**检查：**
1. 报告数据是否完整？
2. URL 参数是否正确编码？
3. JavaScript 是否有语法错误？
